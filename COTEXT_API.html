React Context API

This page provides a detailed explanation of React Context API, including how to share state globally across components, practical examples, and best practices.

1. What is Context API?

The Context API in React is a way to **share state globally** across components without passing props manually at every level (avoiding prop drilling).

  • Useful for global data like theme, authentication, language, or settings.
  • Works with functional components using useContext hook.
  • Lightweight alternative to Redux for simple global state management.

2. Creating a Context

Step 1: Create a context using createContext():

  import React, { createContext } from 'react';
  
  export const ThemeContext = createContext(); // create context

3. Providing Context

Step 2: Wrap the part of your app that needs access to the context with a Provider and pass a value:

  import React from 'react';
  import { ThemeContext } from './ThemeContext';
  import ChildComponent from './ChildComponent';
  
  function App() {
    const theme = 'dark';
  
    return (
      <ThemeContext.Provider value={theme}>
        <ChildComponent />
      </ThemeContext.Provider>
    );
  }
  
  export default App;

4. Consuming Context

Step 3: Access the context value using useContext hook in a child component:

  import React, { useContext } from 'react';
  import { ThemeContext } from './ThemeContext';
  
  function ChildComponent() {
    const theme = useContext(ThemeContext); // get value from context
  
    return <div>Current theme: {theme}</div>;
  }
  
  export default ChildComponent;

5. Example 1: Theme Switcher

A complete example of changing theme globally using Context API:

  import React, { createContext, useState, useContext } from 'react';
  
  const ThemeContext = createContext();
  
  function ThemeProvider({ children }) {
    const [theme, setTheme] = useState('light');
    const toggleTheme = () => setTheme(theme === 'light' ? 'dark' : 'light');
  
    return (
      <ThemeContext.Provider value={{ theme, toggleTheme }}>
        {children}
      </ThemeContext.Provider>
    );
  }
  
  function Child() {
    const { theme, toggleTheme } = useContext(ThemeContext);
    return (
      <div style={{ background: theme==='light'?'#fff':'#333', color: theme==='light'?'#000':'#fff', padding:'20px'}}>
        Current Theme: {theme}
        <button onClick={toggleTheme}>Toggle Theme</button>
      </div>
    );
  }
  
  function App() {
    return (
      <ThemeProvider>
        <Child />
      </ThemeProvider>
    );
  }
  
  export default App;

6. Example 2: Authentication Status

Using Context API to manage global user login state:

  import React, { createContext, useState, useContext } from 'react';
  
  const AuthContext = createContext();
  
  function AuthProvider({ children }) {
    const [user, setUser] = useState(null);
    const login = () => setUser({name:'Ayesha'});
    const logout = () => setUser(null);
  
    return (
      <AuthContext.Provider value={{user, login, logout}}>
        {children}
      </AuthContext.Provider>
    );
  }
  
  function Navbar() {
    const { user, login, logout } = useContext(AuthContext);
    return (
      <div>
        {user ? <div>Hello {user.name} <button onClick={logout}>Logout</button></div>
             : <button onClick={login}>Login</button>}
      </div>
    );
  }
  
  function App() {
    return (
      <AuthProvider>
        <Navbar />
      </AuthProvider>
    );
  }
  
  export default App;

7. Context API Best Practices

  • Use Context only for global or shared state, not local state.
  • Split context into multiple contexts for different concerns (theme, auth, settings).
  • Wrap only components that need access to reduce unnecessary renders.
  • Combine with custom hooks for cleaner logic.
  • Keep context values simple and avoid passing large objects unless necessary.
